Moviendo el blog a GitHub

Desde ahora todo lo que está visible en el blog, está hospedado en GitHub Pages.

Adiós Blogger, fueron buenos años

Decidí mover los 134 artículos del blog (desde el año 2009) a GitHub. Blogger fue un buen aliado pero necesitaba tener mayor control sobre el código. Miré otras opciones como Wordpress, Medium, y Jekyll, pero quise dejar las cosas simples y a la misma vez tener bajo control en 100% el código de todo el blog. Solo usando HTML5, CSS3, y JavaScript.

Limpiando el código y redundancias

Para mover cientos de artículos desde Blogger es un proceso algo largo. Primero hay que limpiar divs y brs por montones. Blogger no es muy semántico y estructurado para armar sus artículos, y tuve que poner orden al código, otorgar significado semántico y no solo llenar de divs, y también eliminar redundancias.

En cada artículo aproveché de usar JavaScript para que use el mismo header en todos, así que cuando decida algún día cambiar el header, solo modifico un archivo JS. Pero para el Inicio, Articulos, y Yo, tuve que dejar el header en el HTML al cambiar el estado del menú nav. En el head hice lo mismo, crear un archivo JS con el meta-data de todas las páginas del blog.

También puse algo de Flexbox en el header para ajustar el modo responsive que había hecho anteriormente. Las tipografías seguirán igual como ya lo había dicho. Y ahora todas las fuentes usan medidas rem. También usé un segundo archivo CSS basandome en Normalize, para que el blog responda bien en los demás navegadores.

Todas las imágenes están optimizadas y redimensionadas para su uso en web. El favicon está en PNG, y no pude usar SVG como favicon porque solo Firefox lo soporta por ahora. Y si, ya revisé el HTML para que validará sin errores en la W3C.

Mejor SEO

La mayoría de las visitas de cualquier sitio provienen desde Google, y al tener un sitio con buena semántica y mucho contenido en texto, benificia al SEO.

Accesibilidad

Ahora todas las imágenes usan alt con descripción de imagen, etiquetas en menús, y titulos individuales de cada página. Todas las páginas indican el idioma en español. También nuevo 404. GitHub Pages usa HTTPS por defecto.
