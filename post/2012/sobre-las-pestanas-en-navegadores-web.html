Sobre las pestañas en navegadores web

En OMG! Ubuntu! vi un artículo sobre por qué las pestañas deben morir, bueno no sé si morir pero mi idea va enfocada a los navegadores web, al acceder a una interfaz multitarea de todos los sitios abiertos como previsualización.

El problema con las pestañas en navegadores web es mantener la limpieza de la interfaz, es demasiado el texto que se acumula en la barra de pestañas, y al abrir muchas se pierde la rápida detección de cual pestaña es la correcta debido al abundante texto, y no hay una previsualización de cada sitio.

Aquel comportamiento también ocurría con el listado de ventanas abiertas en Windows XP y Ubuntu 10.10 hacia atrás, pero eliminaron el texto de sus listados de ventanas y fueron cambiados por lanzadores con iconos y tooltips emergentes (véase Windows 7 y Ubuntu 11.10).

El manejo de varias pestañas abiertas en dispositivos portátiles se puede solucionar con una vista multitarea, visualizar los sitios abiertos como ventanas pero de una forma ordenada con un reconocimiento más especifico para el usuario, previsualización de sus sitios abiertos.

Escritorio

Primero veamos como manejan los entornos de escritorio la vista general. En Ubuntu 12.10 utilizarán una interfaz multitarea Full Spread, para visualizar y manipular las ventanas, y también previsualizar aplicaciones individualmente y crear nuevas instancias de una aplicación desde una vista general (imagen arriba) y no depender tanto de las pestañas de un navegador. En GNOME Shell utilizan una interfaz multitarea de las aplicaciones abiertas. En OS X hacen algo parecido con las ventanas en la interfaz multitarea de Mission Control.

Navegadores en smartphones

Safari para iOS móvil trae una vista semi-multitarea ya que hay que deslizar un sitio a la vez. En Chrome para Android se despliegan los sitios como pestañas verticales y con vista previa de cada sitio. Y Firefox para Android previsualiza los sitios al costado izquierdo.

Navegadores en el escritorio

Firefox en el escritorio intentó algo parecido con Panorama para ver los sitios abiertos pero no lo destacó con el tiempo, era una buena idea pero casi la dejó de lado, a pesar de que las últimas versiones la utilizan como opción. En Gnome Web no quieren depender demasiado de las pestañas, y si utilizarán una vista previa de todos los sitios abiertos.

Navegadores en tablets

En Chrome para tablets y Safari para iPad todavía utilizan pestañas en vez de utilizar una vista multitarea de sitios. Pero Firefox para Android en Tablets si trae una previsualización al costado izquierdo más consistente con el modo multitarea de Android 4.0.

Las pestañas de navegadores son igual que las aplicaciones abiertas en un OS, pero estos se localizan como iconos en un lanzador, pero a la misma vez que exista una vista general de las aplicaciones abiertas, como en Windows 7 con Windows Flip, OS X Lion con Mission Control y el Full Spread que se utilizará en Ubuntu 12.10. Entonces son dos interacciones para una misma función pero presentados al usuario de diferente forma. Listado de pestañas o vista multitarea.

La vista multitarea también está orientada en enriquecer la interacción de ambientes táctiles, ese es el campo en que se puede sacar más provecho. Hay que ser consistente con toda la interfaz en móviles y que mantengan una interacción simple. Hay que recordar nuevamente que antes los OS también listaban las aplicaciones abiertas como pestañas en la barra inferior, pero fueron cambiados solo por iconos y con acceso a una vista general para un manejo rápido, y así eliminar el abundante texto de una barra. El navegador web en dispositivos portátiles como Tablets debe ser consistente con el resto de la interfaz también, y dejar de depender de pestañas como un escritorio clásico.
