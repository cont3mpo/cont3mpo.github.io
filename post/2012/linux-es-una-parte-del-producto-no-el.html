Linux es una parte del producto, no el todo

Estaba leyendo ZDNet sobre por qué Google y Canonical no se refieren mucho a Linux para representar una marca y la percepción del usuario al nombre de su producto, y me acordé de algo. El decir Linux es demasiado amplio, no es especifico, no te estás refiriendo a un OS especifico que el usuario reconozca, el decir Linux es hablar del núcleo.

Viejas costumbres

Todavía veo en sitios la costumbre de poner "disponible en Linux o estará en Linux" ¿Pero a que se refieren? Linux es un núcleo utilizado en un sin fin de software en diferentes plataformas, no solamente el escritorio. ¿Entonces por qué lo siguen diciendo? Por viejas costumbres, Linux se asociaba solo al escritorio, pero ahora Linux se utiliza en dispositivos móviles, electrodomésticos y software industrial.

Decir Linux ahora afecta a un sin fin de software, no solo el escritorio. Por eso ahora se debe ser especifico a la hora de decir "disponible en...", especificar el nombre del producto que afectará.

Linux no es todo el producto

Asociar Linux a un producto no es malo, pero Linux no es todo el producto, es una parte de él, como varias otras partes.

Lo que comentan en ZDNet sobre por qué Google y Canonical no nombran Linux es debido a eso, Linux es solo una parte, tal como lo es la parte de la interfaz, el sistema de ventanas, las aplicaciones, drivers, etc. Y se utiliza el nombre del producto final para juntar todo eso y el usuario aprenda a diferenciar, tal como Android o Ubuntu. Los dos utilizan Linux en su interior, pero Linux no es todo el producto.

Ser más específico

Decir el nombre especifico del producto que afectará una aplicación, dispositivo, etc. Por ejemplo: "La próxima versión de Steam estará disponible en Ubuntu", si hubiese dicho Linux en vez de Ubuntu hubiese afectado a Android, Chrome OS, web OS, smart TVs, electrodomésticos, debido a que todos esos utilizan Linux, pero en realidad no estará disponible para todas esas plataformas. Otro ejemplo: "El próximo Nexus 7 vendrá con Android", si hubiese dicho Linux en vez de Android el usuario no sabría con que asociarlo, por eso ser especifico ahorra confusiones.
