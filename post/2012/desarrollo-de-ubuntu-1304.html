Desarrollo de Ubuntu 13.04 y más allá

Esta semana concluyó el Ubuntu Developer Summit para Ubuntu 13.04. Todo este desarrollo apunta al camino hacia 14.04 LTS, el cual empezó en la versión 12.10, así que el próximo año se verán más cosas. Estos son los puntos que más destacaron en el UDS, aunque siempre pueden aparecer más cosas durante su desarrollo.

Mejoras para juegos

Habrá un equipo de desarrollo dedicado a la experiencia de juegos en Ubuntu, los cuales se enfocarán en mejorar gráficos, audio, procesamiento, etc. OpenGL ES, OpenAL, Compiz y drivers abiertos entran de lleno también en esta labor. Debido a que 13.04 incluirá Linux 3.8, las versiones 3.7 y 3.8 incluirán una gran lista de mejoras en los drivers abiertos de Nvidia, AMD y Intel. Otra cosa también es que los desarrolladores quieren que más juegos utilicen OpenGL ES, debido a que Compiz fue portado a este en la última versión de Ubuntu.

Además, Valve anunció la llegada del Beta interno de Steam para los asistentes del UDS. Valve apuesta muy fuerte en Ubuntu luego que Gabe Newell diera su opinión sobre Windows 8.

Interfaz más rápida

El equipo de desarrollo tras Compiz se enfocará en realizar mejoras de velocidad en la interfaz de usuario Unity en 13.04. Debido a regresiones en test de desempeño de interfaz.

Pensando en móvil

A partir del próximo año se comenzará a pensar en cada aspecto interno de Ubuntu en dispositivos móviles y optimizarlo al máximo, mirando claramente hacia 14.04 LTS. Esto no involucra todavía la creación de una interfaz de usuario para móviles, ya que eso será llevado a cabo por Canonical en el camino hacia 14.04 LTS.

El dispositivo base en que Ubuntu se está testeando es el Nexus 7, debido a que es un buen hardware móvil y de bajo precio. Se utilizará como hardware de referencia para que el núcleo de Ubuntu sea optimizado, esto permitirá una visión de como irá su desarrollo en móviles. Ubuntu para Android está siendo testeado en teléfonos actuales como el Galaxy S3.

SDK para Ubuntu

Uno de los puntos que involucra llevar a Ubuntu en varios dispositivos, es la creación de un kit de desarrollo de aplicaciones SDK. Pero esta idea no verá la luz completamente en 13.04, aunque puede ser anticipada. Pero por ahora solo se planeará una gran lista de componentes que incluirá el SDK, aunque si estará completamente lista para 14.04 LTS.

Diseño

Si, el desarrollo del nuevo set de iconos se está llevando a cabo. Canonical junto con University of Reading llevaron a cabo una investigación sobre identidad visual de la interfaz de Ubuntu, el cual involucra más allá de los temas Ambiance/Radiance, se creará un nuevo tema visual junto con los iconos. Pero no hay fecha de lanzamiento todavía.

También hay planes para brindar más consistencia a los menús de la barra superior, debido a que los menús del lanzador son redondeados y transparentes.

Como punto extra, desde hace un tiempo el diseñador encargado del popular set de iconos Faenza, Matthieu James, fue contratado por Canonical para contribuir con la creación del nuevo set de iconos, junto también con el equipo de diseño de Canonical y University of Reading.

Widgets

La interfaz Unity en 13.04 tendrá soporte para widgets, esto involucra también un API para su desarrollo.

Sin Alphas, un Beta

Se botará el ciclo de desarrollo por Alphas, enfocando todo el desarrollo para el público en un solo Beta y la final para 13.04. Esto permitirá menos presión a los desarrolladores y entrega de nuevas características más largas. Hasta ahora las compilaciones diarias de desarrollo han funcionado muy bien, y también habrá un ciclo de testeo cada dos semanas, además las ISOs diarias también serán testeadas automáticamente.

Entonces solo habrán dos ciclos como objetivo: La beta para el 28 de marzo y la final el 25 de abril de 2013. Los demás sabores de Ubuntu (no mantenidos por Canonical; Xubuntu, Kubuntu), tendrán ciclos de desarrollo independiente con alphas y betas.

Desarrollo más estable con GNOME

El equipo de escritorio quiere apegarse a las versiones estables de GNOME en vez de las versiones inestables, como alphas o betas de GNOME. En 12.10 se generaron problemas debido al ciclo inestable de GNOME y su versión 3.6 (como Nautilus por ejemplo), así que que para 13.04 se trabajará con el canal estable de GNOME.

Ah si, Nautilus se quedará y se actualizará a la última versión con nueva barra superior de herramientas. Por último, GStreamer 1.0 o 1.2 vendrá incluido, eso quiere decir que habrá mejoras en reproducción multimedia.

Misma actualización para todo

En la presentación de inicio del UDS, Mark Shuttleworth indicó que Ubuntu recibirá actualizaciones de seguridad al mismo tiempo para todos los dispositivos en 14.04 LTS. Esos dispositivos serían teléfonos, televisores, tablets, escritorios, servidores y hasta supercomputadores al mismo tiempo.

Wubi rediseñado

A pesar de que la interfaz Wubi para instalar Ubuntu dentro de Windows fue removida, si está disponible para ser descargada aparte. Así que ahora hay planes para rediseñar Wubi en 13.04.

Nuevo proceso para subir aplicaciones

Para 13.04 se espera que los desarrolladores puedan utilizar el nuevo proceso de subida de aplicaciones directo al Centro de software de Ubuntu, en vez de esperar hasta el próximo ciclo. Esto proporciona un modo seguro e independiente para cada desarrollador, además de rápido para obtención al usuario.

ARM 64-bits

Ahora que Linux 3.7 vendrá con soporte nativo para la arquitectura ARM 64 bits, en las siguientes versiones de Ubuntu quieren que esté funcionando en ARM 64 bits (actualmente funciona en ARM 32 bit). Eso si, esto será un desarrollo a largo plazo.

Ubuntu TV

Se compartirá más código entre la interfaz Unity de escritorio con la de TV, como el lanzador, indicadores, tablero, HUD, accesos. Además de nuevos accesos en el tablero como DVR para grabar contenido con MythTV y guías de programación. Así que es probable que se vean cambios en la interfaz de Ubuntu TV.
