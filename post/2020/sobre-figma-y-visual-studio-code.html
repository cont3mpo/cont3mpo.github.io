Sobre Figma y Visual Studio Code

Estoy usando más Figma para prototipar interfaces, y Visual Studio Code para escribir.

De Adobe XD a Figma

Vengo usando Adobe XD hace mucho tiempo, al igual que Atom, y siempre me pasa que voy dejando cosas atrás cuando veo que algo nuevo llega a la cuadra y demuestra que puede ser mejor.

Figma y Adobe XD son multiplataforma, ambos apps de escritorio para macOS y Windows, y es una gran ventaja que tienen por sobre Sketch. Pero Figma empuja aún más esa accesibilidad, y también funciona en web, con ello ni siquiera necesitas instalar un app. Esto permite que personas externas o colaboradores puedan dar sus opiniones a proyectos en curso.

Mútiples proyectos

Figma me permite tener varios proyectos activos en pestañas, es como estar dentro de un navegador mismo, pero en Adobe XD tengo que abrir una nueva ventana para cada uno.

Colaboración

Figma y Adobe XD permiten compartir los proyectos en web, para que se puedan comentar, aunque Figma da más libertad en web para que los puedan editar ahí mismo en el navegador. Esto deja una gran posibilidad para trabajar en equipo, además que todos los cambios suceden en vivo.

Sistema de diseños

Figma y Adobe XD permiten trabajar con sistemas de diseños, y con esto puedo editar elementos para crear UI Kits y sistemas de diseños más ordenados. Esto también tomando en cuenta el sistema "en vivo" que se puede usar por web también.

Memoria y espacio en el OS

Figma utiliza muy poco espacio de instalación, solo 165Mb de espacio de disco en macOS. Versus los 463MB que ocupa Adobe XD. Cuando está corriendo, Figma utiliza en memorio RAM, 48MB, en cambio Adobe XD ocupa 436MB de memoria RAM.

Hay una clara diferencia en uso de espacio en disco y memoria usada, los de Adobe tiene que hacer algo para "alivianar" sus apps. El equipo tras Figma lo está haciendo muy bien acá.

De Atom a Visual Studio Code

Con Atom llevo mucho tiempo, es mi regalón. Pero me he dado cuenta que está muy poco optimizado por ahora, creo que Atom fue creado por el equipo de Github a todo esto.

Extensiones y Temas

Una de las cosas que me gustaron de Visual Studio Code, es la gran cantidad de extensiones que tiene, todo los puedes instalar o desinstalar desde el panel lateral, no es necesario reinstalar para ver cambios al instalar alguna.

Memoria y espacio en el OS

Creo que con esto termino de convencerme, VS Code ocupa en espacio de disco 249MB. Versus los 620MB que ocupa Atom. Cuando está corriendo VS Code, utiliza en RAM unos 46MB, en comparación Atom utiliza 97MB de RAM.

La diferencia es abismal, Atom consume demasiado espacio en disco y memoria. Los de Microsoft están haciendo apps muy buenas este último tiempo.
